Definitions

Studio Name

Name of the screen you entered in ScreenCloud Studio

Device Name

Autogenerated name combining the location ID, location name, placement within location, and network name

Device Category

Category that best describes the type of device that is displaying ad content

Model

Description of the device displaying ad content (e.g. Vengo Vending Kiosk)

Screen Size

Diagonal dimension of the screen in inches

Location ID

Your organization's internal ID for the place of business where the screen is located (e.g. 12345)

Location Name

The name of the place of business where the screen is located (e.g. Wal-mart)

Placement within Location

The area within the place of business where the screen is placed (e.g. Front Entrance, Aisle 1 Endcap)

Instructions
Enable Vengo access

Vengo needs access to your ScreenCloud account to pull the information required to onboard your screens to the ad server.

Instructions on how to generate an API key/token can be found here. When selecting permissions, check:

  • App
  • App Instance
  • Media
  • Playlist
  • Screen
Use default parameters

If you are onboarding multiple screens of the same type, use the "Default Parameters" form to quickly copy the same parameters to multiple screens

Screens

Expand each section to find a set of screens

  • Online, Not Onboarded - screens that are eligible to be onboarded to Vengo's adserver
  • Online, Onboarded - screens that are already onboarded to Vengo's adserver
  • Offline, Not Onboarded - screens that are already onboarded to Vengo's adserver but are currently offline
  • Offline - screens that are currently offline

Click the Edit button next to the Studio name to modify each parameter and add the screen to the onboard list

Click the pin next to the address to modify the address

Click Copy from Default to pull down the selections from the Default Parameters form

Click Reload from Studio if you've made a change in ScreenCloud studio and need to refresh the data

Click Save and Add to List when all selections are made and the screen is ready for onboarding

Submit screens to onboard

After generating the list of screens to onboard, click Submit to transfer the data to Vengo
